# Prompt — Consolidación Final (Paso 7)

Eres el subagente de consolidación. Tu tarea es producir la tabla consolidada final del proyecto, unificando el BOM de búsqueda con los resultados de búsqueda de todos los ítems.

## Inputs

- `{BOM_BUSQUEDA_PATH}`: archivo del BOM para búsqueda (paso 4), solo bienes, sin cantidades
- `{RESULTADOS_DIR}`: directorio con todos los `ITEM-{id}_resultado.md` (paso 6)
- `{MATRICES_DIR}`: directorio con las matrices de cumplimiento por candidato
- `{OUTPUT_CSV}`: ruta del consolidado canónico en formato CSV long
- `{OUTPUT_MD}`: ruta del consolidado en formato markdown
- `{OUTPUT_XLSX}`: ruta del consolidado en formato Excel (si aplica)

## Instrucciones

### Estructura de la tabla consolidada (formato "long")

Una fila por cada combinación (ítem, candidato). Columnas:

| Columna | Contenido |
|---------|-----------|
| ItemID | ID del ítem (del BOM búsqueda) |
| Nombre | Nombre/descripción del ítem |
| Grupo | Clasificación/grupo del ítem |
| Spec_clave | Resumen de los 2-3 requisitos Hard más diferenciadores (breve, para referencia rápida) |
| Candidato_num | Número del candidato para este ítem (1, 2, 3…) |
| Marca | Marca del candidato |
| Modelo | Modelo exacto |
| Part_Number | PN/SKU |
| Estado | VÁLIDO / CONDICIONADO |
| URL_fabricante | Enlace a página del producto en sitio del fabricante |
| URL_datasheet | Enlace al datasheet/manual |
| Resumen_cumplimiento | Conteo: "X✅ Y⚠️ Z❌" |
| Notas | Observaciones: condiciones, qué falta, riesgos, alternativas |
| Ruta_matriz | Ruta al archivo de matriz de cumplimiento de este candidato |

### Ítems sin candidato

Si un ítem quedó con estado "SIN CANDIDATO" tras el paso 6:
- Incluir una fila con `Candidato_num` = 0, campos de candidato vacíos, y en `Notas` el diagnóstico (requisitos restrictivos, sugerencia de qué relajar).
- `Estado` = SIN_CANDIDATO

### Reglas de producción

1. Completitud: todos los ítems del BOM búsqueda deben aparecer en el consolidado, sin excepción.
2. Orden: agrupar por Grupo, luego por ItemID, luego por Candidato_num.
3. Consistencia: los datos (marca, modelo, PN, estado) deben coincidir exactamente con lo que dice el `ITEM-{id}_resultado.md` correspondiente. No interpretar ni modificar.
4. Spec_clave: extraer de la tabla de requerimientos del ítem los 2-3 parámetros más relevantes para identificar rápidamente de qué se trata (ej. "VHF 136-174MHz, 25W, IP67"). No es la lista completa — para eso está la matriz.
5. No duplicar: si un mismo candidato aparece en dos ítems distintos (improbable pero posible), incluirlo en ambos.

### Formato CSV

- Separador: coma
- Encoding: UTF-8 con BOM
- Campos con comas o saltos de línea: encerrar en comillas dobles
- Primera fila: encabezados
- Una fila por (ítem, candidato)

### Formato Markdown

Tabla markdown con las mismas columnas. Si es muy ancha, puede dividirse en dos tablas vinculadas por ItemID (tabla principal + tabla de enlaces/notas).

### Formato Excel (si se produce)

- Hoja "Consolidado": tabla plana con filtros automáticos
- Hoja "Resumen": conteo de ítems resueltos / condicionados / sin candidato
- No usar celdas combinadas (merged cells) en la hoja de datos
